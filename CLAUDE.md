## 현재 상태 (중요)
이 저장소는 아직 코드가 작성되지 않은 초기 단계입니다:
- `src/main.py`, `requirements.txt` 는 둘 다 빈 파일입니다. 아직 어떤 의존성이나 구현도 존재하지 않습니다.
- 빌드/린트/테스트 도구가 전혀 설정되어 있지 않습니다. `pytest`, `flake8` 등 존재를 가정하지 말고, 먼저 실제로 있는지 확인하세요.
- `docs/`에는 조원별 2주차 과제 PDF만 있으며, 코드나 설계 문서가 아닙니다.
- git 저장소로 초기화되어 있지 않습니다 (`git init`이 필요할 수 있음).

이런 상태이므로, 새 코드를 작성할 때는 기존 컨벤션을 따르기보다 아래 로드맵에 맞춰 구조를 새로 잡아야 합니다.

## 개발 계획 및 일정 (로드맵)
- 1~2주차: 아키텍처 설계 및 정적 분석 자동화 스크립트(JADX CLI 연동) 구현
- 3~4주차: 동적 분석 모듈(Frida) 개발 및 Python 연동
- 5~6주차: 네트워크 패킷 파싱 로직 및 전체 파이프라인 통합
- 7~8주차: Streamlit 기반 웹 대시보드 UI 구현 및 검증

로드맵에 따라 앞으로 `src/`에는 대략 다음과 같은 구성 요소가 추가될 것으로 예상됩니다: JADX CLI를 감싸는 정적 분석 래퍼, Frida 기반 동적 계측 모듈(Python 바인딩), 네트워크 패킷 파싱 컴포넌트, 이들을 하나로 묶는 파이프라인, 마지막으로 Streamlit UI. 각 단계에서 필요한 의존성(예: `frida`, `streamlit`, `scapy` 등)이 추가되면 `requirements.txt`도 함께 갱신해야 합니다.

## 디렉터리 구조
- `src/`: 파이썬 소스 코드 위치 (`main.py`가 엔트리 포인트)
- `docs/`: 조원별 분석 자료 및 과제 PDF 보관 (편집 대상 아님)
- `requirements.txt`: 의존성 패키지 목록

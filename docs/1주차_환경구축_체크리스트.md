이 문서는 오늘 22시 중간점검용 과제를 실제로 무엇을 하라는 것인지 단계별로 풀어쓴 것이다. 크게 두 부분으로 나뉜다.

- **A. 개발환경 세팅** — 각자 자기 컴퓨터에서 직접 설치/실행해야 하는 부분 (GUI 설치 과정이라 본인 PC에서만 가능)
- **B. 프로젝트/설계** — 문서·스키마·다이어그램 작업 (이 부분은 이번에 대신 초안을 만들어 `docs/`, `schemas/`에 저장해 두었음)

## A. 개발환경 세팅 (각자 로컬 PC에서 직접 수행)

### 1. Android Studio 설치
- https://developer.android.com/studio 에서 설치 프로그램 다운로드 후 설치.
- 설치 시 "Android SDK", "Android SDK Platform", "Android Virtual Device" 항목이 함께 체크되어 있는지 확인 (기본값으로 대부분 포함됨).

### 2. AVD(가상 기기) 생성
- Android Studio 실행 → 상단 메뉴 또는 시작 화면에서 **Device Manager** (예전 버전은 AVD Manager) 열기.
- "Create Device" → 기기 종류 선택 (Pixel 계열 아무거나 무난).
- **시스템 이미지 선택이 이번 과제에서 가장 중요한 부분**:
  - **Google APIs** 이미지를 선택할 것 (Google Play 이미지 아님).
  - 이유: Google Play 이미지는 시스템 파티션이 읽기 전용(read-only)이라서, 3~4주차에 `frida-server`를 시스템에 넣으려고 할 때 막힘. Google APIs 이미지는 루팅(root) 권한 획득이 가능해서 이 문제가 없음.
  - **API 레벨은 28~31 사이**로 선택 (예: API 30). 너무 최신 버전(API 33 이상)은 피할 것 — 루팅/후킹 관련 이슈가 더 많고, 실제 뱅킹 악성코드들이 주로 타겟팅하는 범위도 이 언저리임.
- 생성 완료 후 한 번 GUI로 실행해서 정상 부팅되는지만 확인.

### 3. SDK 커맨드라인 도구 PATH 등록
Android Studio가 설치한 SDK 경로 안에 있는 다음 폴더들을 PATH 환경변수에 추가:
- `<SDK 경로>/platform-tools` (adb 포함)
- `<SDK 경로>/emulator` (emulator 실행파일 포함)
- `<SDK 경로>/cmdline-tools/latest/bin` (sdkmanager, avdmanager 포함)

SDK 경로는 보통 Android Studio의 Settings → Languages & Frameworks → Android SDK 화면에서 확인 가능. Windows는 시스템 환경변수, Mac/Linux는 `~/.zshrc` 또는 `~/.bashrc`에 `export PATH=...` 추가.

이 작업을 하는 이유: 지금부터는 에뮬레이터를 GUI 클릭이 아니라 **터미널 명령어로 스크립트가 직접 띄우는 방식**을 쓸 것이기 때문. 자동화 파이프라인(6주차 오케스트레이터)에서 결국 Python이 `subprocess`로 이 명령어들을 호출하게 됨.

### 4. 헤드리스 실행 검증 (이번 과제의 핵심 확인 사항)
터미널에서 아래 순서로 실행:

```bash
# 1) 만들어둔 AVD 이름 확인
emulator -list-avds

# 2) 창 없이(headless) 실행
emulator -avd <AVD이름> -no-window

# 3) 다른 터미널 창에서 인식 확인
adb devices
```

`adb devices` 결과에 기기가 `device` 상태로 뜨면 성공. 이게 안 되면 이후 모든 주차 작업(3주차 Frida, 8주차 실제 샘플 테스트 등)이 다 막히므로, 오늘 반드시 여기까지 확인해야 함.

### 5. adb / jadx-cli / apktool 설치 및 버전 확인
```bash
adb version
jadx --version
apktool -version
```
셋 다 에러 없이 버전이 출력되면 됨. jadx는 https://github.com/skylot/jadx 릴리즈에서, apktool은 https://ibotpeaches.github.io/Apktool/ 안내대로 설치.

### 6. 정상 APK로 전체 연동 확인
아무 정상 APK(구글 플레이에서 받은 apk 등) 1~2개를 준비해서:
```bash
adb install <파일경로>.apk
```
설치가 성공하면 SDK + 에뮬레이터 + adb 전체 연동이 확인된 것.

아래 항목은 이번에 대신 만들어서 저장소에 넣어 두었다. 팀원들과 확인 후 필요하면 수정하면 된다.

### 아직 팀 논의가 필요한 것

- **GitHub 저장소 연결 / 브랜치 전략**: 팀 git 저장소는 이미 있지만 아직 이 폴더와 연결하지 않은 상태 (CLAUDE.md에 명시됨). 브랜치 전략은 팀 전체가 합의해야 하는 부분이라 임의로 정하지 않았음. 아래는 흔히 쓰는 무난한 안이니 참고해서 오늘 회의에서 확정하면 됨:
  - `main`: 항상 동작하는 상태만 유지
  - `dev`: 각자 작업을 합치는 브랜치
  - `feature/static-analyzer`, `feature/dynamic-agent`, `feature/network-analyzer`, `feature/dashboard`: 모듈별 작업 브랜치
- 연결 방법이 결정되면, 이 폴더에서 `git init` → `git remote add origin <팀 저장소 주소>` → `git add . && git commit -m "1주차: 환경 구축 및 아키텍처/스키마 초안"` → `git push` 순서로 진행하면 됨.

## 오늘 22시까지 각자 확인해야 할 최소 기준

1. `emulator -avd <이름> -no-window` 실행 → `adb devices`에 기기 인식됨
2. `adb version`, `jadx --version`, `apktool -version` 모두 정상 출력
3. 정상 APK 1개 `adb install` 성공
4. 위 3가지를 팀 채팅/문서에 스크린샷 또는 로그로 공유
